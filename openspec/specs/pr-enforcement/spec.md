# pr-enforcement

## Purpose
プルリクエスト（PR）の品質とプロセス遵守を機械的に検証し、適切な状態でのマージを保証します。これにより、レビュープロセスの効率化と人的ミスの削減を図り、開発サイクル全体の品質を向上させることを目的とします。

## Requirements

### Requirement: PRのTODO完了の強制
システムは、プルリクエストの本文に含まれる全てのチェックボックスが完了状態であることを検証しなければならない（MUST）。

#### Scenario: TODOが未完了の場合
- **WHEN** ユーザーが未チェックのチェックボックス（`- [ ]`）を含むPRを作成または編集する
- **THEN** ステータスチェックが失敗し、TODOが未完了であることを通知する

#### Scenario: 全てのTODOが完了している場合
- **WHEN** ユーザーが全てのチェックボックスを完了（`- [x]`）にする、またはTODOリストを含まないPRを作成・編集する
- **THEN** ステータスチェックが成功する
