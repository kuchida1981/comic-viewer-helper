## Context

「I'm feeling lucky」機能のランダム遷移で、非公開作品へのジャンプが発生している。非公開作品の判定ルールはサイト固有であるため、アダプター層で吸収し、`isPrivate` フラグとして関連作品データに付与する。フラグを付与した後、遷移ハンドラでのみフィルタリングを行い、Info モーダルでの表示には影響しない。

## Goals / Non-Goals

**Goals:**
- アダプター層で非公開作品を判定し、`isPrivate` フラグを付与する
- ランダム遷移の抽選対象から非公開作品を除外する

**Non-Goals:**
- Info モーダルにおける非公開作品の表示を変更する
- 🎲 ボタンの表示・非表示を非公開フィルタリング結果で切り替える（レイアウト安定のため）

## Decisions

### 判定ロジックの配置箇所：アダプター層
- **Why**: 非公開の判定基準（タイトル先頭が「非公開」）はサイト固有のルールであるため、アダプターが最も適切な場所
- **Alternatives**: UIManager で判定 → ビジネスロジックがUI層に混入するため不適

### フィルタリング箇所：`onLucky` ハンドラ内のみ
- **Why**: Info モーダルでは非公開作品も表示し続ける要件がある。Storeの `relatedWorks` にフィルタリングを適用すると全箇所に影響が出る
- **Alternatives**: Store 保存時にフィルタリング済みリストを別途保持 → 単一要件向けに過度な複雑さ

### `hasRelated` の判定は変更しない
- **Why**: 全作品が非公開の場合は実現不能なエッジケースであり、レイアウト変動を避けるが優先される

## Risks / Trade-offs

- [リスク] 今後非公開判定基準が変わった場合、アダプターの修正が必要
  → アダプター層に閉じているため、変更コストは最小
- [トレードオフ] 全作品が非公開の場合、🎲 を押しても遷移しない
  → 実現性が極めて低いエッジケースであり、レイアウト安定を優先
