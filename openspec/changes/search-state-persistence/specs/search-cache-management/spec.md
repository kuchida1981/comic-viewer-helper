## ADDED Requirements

### Requirement: 検索結果のキャッシュ管理
システムは、検索キーワードごとに検索結果と取得日時をキャッシュとして保持し、一定期間（デフォルト1時間）有効なものとして管理しなければならない (SHALL)。

#### Scenario: キャッシュの有効期限内での利用
- **WHEN** 検索が実行され、有効期限内のキャッシュが存在する時
- **THEN** APIリクエストを行わず、キャッシュされた結果を即座に表示すること

#### Scenario: キャッシュ期限切れ時のSWR更新
- **WHEN** 検索が実行され、キャッシュが存在するが有効期限が切れている時
- **THEN** キャッシュされた結果を即座に表示した上で、バックグラウンドで最新データを取得し、完了後に結果を更新すること

### Requirement: キャッシュの永続化
検索結果のキャッシュは、ページ遷移後も利用できるように永続化されなければならない (SHALL)。

#### Scenario: ページ遷移を跨いだキャッシュの復元
- **WHEN** ページ遷移後に検索モーダルを開き、前回と同じキーワードで検索が要求された時
- **THEN** `localStorage` から復元されたキャッシュを利用して結果が表示されること
