## Context

現在、`NavigationButtons.js` を通じて基本的なページ操作ボタン（最初/最後、前/次、Info、Help）を提供しています。今回の変更では、これらに加えて、関連作品からランダムに1つを選んで遷移する「おすすめ（ランダム）」ボタンを追加します。この機能は、読了後の回遊性を高めることを目的としています。

## Goals / Non-Goals

**Goals:**
- UIコンテナに新しいボタン（🎲）を追加する。
- 関連作品が存在する場合にのみボタンを有効/表示にする。
- クリック時に `metadata.relatedWorks` から公平にランダムな1つを選択して遷移する。
- 多言語対応（日本語/英語）を提供する。

**Non-Goals:**
- 関連作品自体の抽出ロジックの変更（既存の `DefaultAdapter` の成果を利用する）。
- 他のサイトへの遷移制限の緩和（あくまで `relatedWorks` にあるURLのみ）。

## Decisions

### 1. UIコンポーネントの拡張
`NavigationButtons.js` の `createNavigationButtons` 関数を拡張し、`onLucky` コールバックを受け取るようにします。
- **理由**: 既存のナビゲーションボタン群とデザインを統一し、DOM管理を一元化するため。

### 2. ランダム遷移ロジックの配置
ランダムな要素の選択と遷移（`window.location.href` の更新）は、`UIManager.js` の `updateUI` 内で `onLucky` コールバックとして定義します。
- **理由**: 遷移は外部URLへの移動を伴う副作用であり、UIイベントのハンドリングとして `UIManager` で扱うのが現状の設計（`onInfo`, `onHelp` 等）と一貫しているため。

### 3. ボタンの表示制御
`metadata.relatedWorks` が空の場合、ボタン要素自体を作成しないか、または `display: none` で隠します。
- **理由**: ユーザーが無効なボタンをクリックして混乱するのを防ぐため。

### 4. アイコンの選択
ボタンのテキスト（アイコン）には Emoji の「🎲」を使用します。
- **理由**: 現在のボタンがテキストベース（`<<`, `>`, `?` 等）で構成されており、Emoji は軽量で直感的なため。

## Risks / Trade-offs

- **[Risk] メタデータの不整合** → `DefaultAdapter` が正しく抽出できていない場合、空リストになるか壊れたURLになる可能性がある。
  - **Mitigation**: 遷移前にURLの存在チェックを行い、空の場合は遷移処理をスキップする。
- **[Trade-off] 遷移の即時性** → ランダム遷移時に確認ダイアログを出さない。
  - **Rationale**: 「I'm feeling lucky」の性質上、即時性が楽しさに繋がるため。
