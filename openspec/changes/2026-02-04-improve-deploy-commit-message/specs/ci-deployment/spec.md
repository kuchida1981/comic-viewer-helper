## ADDED Requirements

### Requirement: デプロイコミットメッセージの詳細化
デプロイワークフローは、配布用ブランチへのコミットメッセージに、トリガーとなった元コミットの情報とデプロイメタデータを含めなければならない (SHALL)。

#### Scenario: デプロイコミットメッセージの構成
- **WHEN** デプロイワークフローが実行され、配布用ブランチに成果物をコミットする
- **THEN** そのコミットメッセージは以下を含む:
  - トリガーコミットのサブジェクト（件名）
  - トリガーコミットのボディ（説明）
  - デプロイメタデータ（リポジトリ名、コミットSHA、実行者、イベント種別）

#### Scenario: トリガーコミットメッセージが空の場合
- **WHEN** トリガーコミットのメッセージが空であった場合
- **THEN** サブジェクト・ボディとして空文字列を適切に扱い、メタデータ部分は正常に含まれる
