## ADDED Requirements

### Requirement: テストカバレッジの自動計測
システムは、テスト実行時にコードの網羅率（カバレッジ）を自動的に計測し、レポートを生成しなければならない。

#### Scenario: カバレッジレポートの生成
- **WHEN** `npm test`（またはカバレッジ用コマンド）が実行される
- **THEN** `dist/coverage` などのディレクトリに HTML 形式またはテキスト形式のレポートが出力されること

### Requirement: 静的型チェックの実施
システムは、実行コードを変更することなく、JSDoc と型定義に基づいてコードの整合性を静的に検証できなければならない。

#### Scenario: 型エラーの検知
- **WHEN** 関数に対して誤った型の引数を渡すコードが存在する状態で型チェックコマンドを実行する
- **THEN** エラーメッセージが表示され、終了コードが非 0 になること

### Requirement: UserScript メタデータの検証
システムは、UserScript のヘッダー情報（メタデータ）が規約に従っているか、および論理的な誤りがないかを自動的に検証しなければならない。

#### Scenario: 無効なメタデータの検知
- **WHEN** `@match` 等に無効な URL パターンが含まれている
- **THEN** リンターによってエラーとして指摘されること
