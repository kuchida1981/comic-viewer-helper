## Context

現状、`src/main.js` に未使用関数 `getCurrentPageIndex` が残っており、`npm run lint` を実行すると警告が表示される。
また、ESLint のデフォルト設定では警告が発生しても終了コードは 0（成功）となるため、CI プロセスを通過してしまう。

## Goals / Non-Goals

**Goals:**
- 未使用関数の削除による Lint 警告の解消。
- 警告が一つでもある場合に Lint コマンドを失敗させる設定の導入。
- CI プロセスにおいて、「警告ゼロ」をマージ条件として強制。

**Non-Goals:**
- 既存の ESLint ルール自体の変更（ルールは維持したまま、警告をエラーとして扱う）。
- 他の警告（もしあれば）の自動修正（今回は明示的に指摘されているものに対応）。

## Decisions

- **Decision 1: `--max-warnings 0` オプションの採用**
    - 理由: ESLint の標準機能であり、追加の依存関係なしに「警告ゼロ」を強制できるため。
- **Decision 2: 既存の `npm run lint` スクリプトの書き換え**
    - 理由: ローカル開発時と CI 時で挙動を統一し、開発者がプッシュ前に警告に気づけるようにするため。

## Risks / Trade-offs

- **[Risk] 些細な警告で CI が落ちるようになる**
    - → [Mitigation] プロジェクトの規模が小さいため、都度修正することでクリーンな状態を保つコストは低いと判断。
