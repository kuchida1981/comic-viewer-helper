## Context

現在のGUI（ナビゲーションパネル）は `position: fixed` で表示され、ドラッグ可能です。しかし、画面端（特に右端や下端）にドラッグすると、ブラウザのレイアウトエンジンによってコンテナの幅が縮小され、内部のコンテンツが折り返されてしまう現象が発生しています（Issue #16）。

## Goals / Non-Goals

**Goals:**
- 画面端に配置しても、GUIパネルのレイアウト（ボタンの並び、サイズ）が維持されること。
- テキストやボタンが折り返されないこと。

**Non-Goals:**
- 画面外へのドラッグ移動の完全な制御（今回はCSSによる表示崩れ修正にフォーカスし、JSによる座標クランプはスコープ外とする）。

## Decisions

### CSSによるレイアウト固定
`src/main.js` の `createNavigationUI` 関数内で、GUIコンテナのスタイル定義に以下のプロパティを追加します。

1.  **`flex-wrap: nowrap`**: Flexコンテナ内のアイテムが改行されないようにします。
2.  **`white-space: nowrap`**: テキストコンテンツが改行されないようにします。
3.  **`width: max-content`**: コンテナの幅をコンテンツの最大幅に合わせ、親要素（ウィンドウ）の幅による圧縮を防ぎます。

### JSによる座標制御の見送り
ドラッグ時に画面外にはみ出さないようにJSで座標を制限する方法もありますが、今回のIssueの本質は「表示崩れ」であるため、CSSによる対策を優先し、JS変更による副作用（操作感の変化など）を避けます。

## Risks / Trade-offs

- **Risk**: 画面外に完全に移動させてしまった場合、操作できなくなる可能性がある。
    - **Mitigation**: 今回の変更では既存の挙動（画面外への移動可）は変わらない。ユーザーはドラッグで戻すことができる。また、次回ロード時の位置復元ロジックには「画面外チェック」が既に存在するため、再ロードすれば復帰可能。
