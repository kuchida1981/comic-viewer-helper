## ADDED Requirements

### Requirement: ページ番号による直接ジャンプ
システムは、ユーザーが指定したページ番号に直接スクロールする機能を提供しなければならない（SHALL）。

#### Scenario: 正常なページジャンプ
- **WHEN** ユーザーが入力フォームに有効なページ番号（例: 5）を入力し、Enterキーを押す
- **THEN** ビューポートが5枚目の画像の先頭位置までスクロールする
- **AND** 入力フォームのフォーカスが外れる（blur）

### Requirement: ページ番号のバリデーション
システムは、入力されたページ番号が有効な範囲内（1から総画像数まで）であることを保証しなければならない（SHALL）。

#### Scenario: 範囲外の数値入力（過大）
- **WHEN** ユーザーが総ページ数を超える数値（例: 999 / 総20ページ）を入力してEnterキーを押す
- **THEN** ジャンプは実行されず、入力フォームの表示が元の（現在の）ページ番号に戻る
- **AND** ユーザーにエラー（例: 入力欄を一瞬赤くするなど）を通知する

#### Scenario: 数値以外の入力
- **WHEN** ユーザーが数値以外の文字列を入力してEnterキーを押す
- **THEN** ジャンプは実行されず、入力表示が現在のページ番号に戻る
